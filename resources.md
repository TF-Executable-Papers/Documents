# Resourcelist

## Journals and Publishing Houses that Publish Executable Papers

- [Journal for Digital History](https://journalofdigitalhistory.org) - Journal from the field of history, that publishes executable papers in both, a 'layered' version with configurable visibility of different componentes and as a jupyter notebook.
- [Rzine](https://rzine.fr/) - Publishing service that supports and accompanies the publication of executable papers in R by means of _Jupyter Notebooks_ from the arts and humanities.
- [eLife](https://elifesciences.org/collections/d72819a9/executable-research-articles) - Journal from the life sciences that publishes executable papers with code block and inline executable par
- [ReScience C](http://rescience.github.io/) - Journal that reviews (re-)publishes computational research in PDF and code form.
- [The Journal of Open Source Software](https://joss.theoj.org/) - Journal that publishes open source software with article side-by-side.
- [Computo](https://computo.sfds.asso.fr/) - Journal of the _French Statistical Society_ that publishes executable papers in the area of computational/algorithmic statistics.
- [Papers with Code](https://paperswithcode.com/) - Project that publishes research articles together with jupyter notebooks that can be executed in a Binder environment from the machine learning field.

## Guidelines, Standards and Best-Practices Relevant to the Publication of Executable Papers

- [Submission Guidelines 'Fabrikation von Erkenntnis'](https://gitlab.uni.lu/melusina/vdhd/guidelines) - Guidelines for the submission of executable papers to Melusina Press part of the volume 'Fabrikation von Erkenntnis', hosted during the vDHd'21, and co-published with the Zeitschrift für digitale Geisteswissenschaften (ZfdG).
- [Berkely Reproducible Research Practices](https://guides.lib.berkeley.edu/reproducibility-guide) - Guidelines by the _Berkeley University Library_ that teaches how to organize digital research projects in a way that allows reproducibility.
- [A Guide to Reproducible Code in Ecology and Evolution](https://colauttilab.github.io/Readings/BES-Reproducible-Code.pdf) - Guidelines by the _British Ecological Society_ that teaches how to organize digital research projects in a way that allows reproducibility.
- [Code Refinery: Reproducible Research](https://coderefinery.github.io/reproducible-research) - Guidelines by the Code Refinery project that teaches how to organize digital research projects in a way that allows reproducibility.
- [Embed Jupyter Notebook into static webpages locally](https://medium.com/analytics-vidhya/embed-jupyter-notebook-into-static-webpages-locally-79e8a6f9f987) - Tutorial that describes how to embed executable codecells into static websites
- [Security in notebook documents](https://jupyter-notebook.readthedocs.io/en/stable/security.html#security-in-notebook-documents) - Overview about security considerations for using Jupyter Notebooks at the heart of self-hosted executable papers.
- [RMarkdown](https://rmarkdown.rstudio.com) - A format and environment to author executable papers in plain text
- [Pandoc Markdown](https://pandoc.org/MANUAL.html#pandocs-markdown) - A format and environment to author executable papers in plain text
- [Julynter](https://github.com/dew-uff/julynter) - Linter for _Jupyter Notebooks_ in order to provide a consistently written Notebook.
- [nblint](https://github.com/alexandercbooth/nblint) - Linter for _Jupyter Notebooks_ in order to provide a consistently written Notebook.
- [Digitales Konkordanzformat](https://www.mpiwg-berlin.mpg.de/de/aktuelles/brill-und-max-planck-institut-fuer-wissenschaftsgeschichte-unterzeichnen-vereinbarung) - not yet public attempt from Brill and the Max-Planck-Institute for the History of Knowledge to publish text, data and processes in the field of Humanities.

## Software and Services that were Used or are Usable for Services and Infrastructure of Executable Papers

- [Executable Books Project](https://executablebooks.org) - Project that builds and develops around _Jupyter Notebooks_ in order to provide online publishable executable books.
- [Stencila](https://stenci.la/) - Proprietary software environment for the authoring and packaging of executable papers that is used by the _eLife_-journal.
- [Jupyter Books Project](https://jupyterbook.org) - Project that builds and develops around _Jupyter Notebooks_ in order to provide static book files that meet the sophisticated formatting requirements and other affordances of the scholarly publishing sector.
- [Thebe](https://github.com/executablebooks/thebe) - JavaScript library that enables to turn static HTML pages into interactive documents with the help of Binder.
- [Binder](https://mybinder.org/) - Infrastructure and service that enables to run a Jupyter Notebook, stored in a git-repository, in the cloud.
- [GESIS Notebooks](https://notebooks.gesis.org/about.html) - Service by the _Leibniz Institute for the Social Sciences_ that uses the _Binder_-infrastructure to run _Jupyter Notebooks_ in the cloud.
- [Jupyter Notebook Publishing](https://myst-nb.readthedocs.io/en/latest/) - A Sphinx and Docutils extension for compiling Jupyter Notebooks into high quality documentation formats.

## Executable Papers Examples

- [Introduction to Cultural Analytics & Python](https://melaniewalsh.github.io/Intro-Cultural-Analytics/welcome.html) - Teaching book in HTML, PDF and as a Jupyter Notebook, created with the _Executable Book Project_, allowing execution in the 'static' HTML version.
- [Fabrikation von Erkenntnis](https://www.melusinapress.lu/projects/fabrikation-von-erkenntnis) - Volume by Melusina and ZfdG as a companion to the vDHd conference, providing HTML, PDF and Jupyter Notebook versions of Executable Papers and allowing execution and modification of code in the 'static' HTML version by means of the _Thebe_ library.
- [The Turing Way](https://the-turing-way.netlify.app) - Teaching book for making data science accessible, created with the _Executable Book Project_, allowing execution in the 'static' HTML version.
- [Introduction to Python for Computational Science and Engineering](https://fangohr.github.io/introduction-to-python-for-computational-science-and-engineering/) - Teaching book in HTML, PDF and as a Jupyter Notebook, created with the _Executable Book Project_, allowing execution in the 'static' HTML version.
- [GWOSC Tutorials](https://github.com/minrk/ligo-binder) - Learning resource of the Gravitational Wave Open Science Center from the field of Astronomy, using Jupyter Notebooks that can be launched, using Binder, Colab or Azure.
